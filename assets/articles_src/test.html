Задание 27 — это задание с развёрнутым ответом, в котором нужно написать программу. Оно стоит максимум 4 балла и является последним и самым сложным заданием на экзамене. Многие сдают ЕГЭ по информатике, вообще не трогая эту задачу. Но с правильной подготовкой её можно решить стабильно и быстро.

📌 Важно знать
Задание 27 проверяется вручную экспертами. Частичный ответ тоже оценивается — можно получить 1–3 балла за неполное, но рабочее решение. Не оставляйте задание пустым.

Что проверяет задание

В задании 27 всегда нужно написать программу, которая читает данные, обрабатывает их по заданному условию и выводит результат. Ключевые умения, которые здесь проверяются:

  • Чтение входных данных из файла или стандартного ввода
  • Работа с последовательностями, строками, массивами
  • Реализация перебора, поиска, сортировки
  • Корректная обработка граничных случаев
  • Вывод результата в точном соответствии с условием

Язык программирования — на ваш выбор: Pascal, Python, C++, Java, C#. На практике абсолютное большинство сдающих выбирают Python — он короче, читаемее и прощает больше ошибок.

Структура типичной задачи

Все задачи задания 27 делятся на несколько устойчивых типов. Разберём каждый из них.

Тип 1 — Перебор и подсчёт

Дана последовательность чисел. Нужно найти все элементы, удовлетворяющие условию, и что-то с ними сделать — посчитать, найти максимум, сумму и т.д.

Дан файл с целыми числами. Найдите количество пар соседних элементов, у которых оба числа являются чётными, и выведите максимум среди таких пар. — Типовое задание ФИПИ, вариант 2026

Шаблон решения для этого типа:

Python
with open('input.txt') as f: data = list(map(int, f.read().split())) count = 0 max_val = None for i in range(len(data) - 1): a, b = data[i], data[i + 1] if a % 2 == 0 and b % 2 == 0: count += 1 pair_max = max(a, b) if max_val is None or pair_max > max_val: max_val = pair_max print(count, max_val) # оба значения в одну строку
⚡ Лайфхак
Всегда добавляйте обработку случая, когда подходящих пар нет. Эксперт обязательно проверяет граничные входные данные — именно на них чаще всего «сыплются» решения.

Тип 2 — Работа со строками

Дана строка или набор строк. Нужно найти подстроки по условию, посчитать символы, найти самое длинное/короткое слово и т.д. В Python это решается особенно элегантно:

Python
with open('input.txt', encoding='utf-8') as f: lines = f.readlines() result = [] for line in lines: words = line.split() for word in words: # пример: слова, начинающиеся на гласную if word[0].lower() in 'аеёиоуыэюя': result.append(word) print(len(result)) print(max(result, key=len))

Тип 3 — Рекурсия и динамика

Самый сложный подтип. Встречается реже, но именно он отделяет 3 балла от 4. Здесь нужно уметь строить рекуррентное соотношение и либо реализовать рекурсию с мемоизацией, либо динамическое программирование снизу вверх.

Python — динамика
from functools import lru_cache # Рекурсия + автоматическая мемоизация @lru_cache(maxsize=None) def f(n): if n <= 1: return n return f(n - 1) + f(n - 2) # sys.setrecursionlimit нужен для глубокой рекурсии import sys sys.setrecursionlimit(100000) n = int(input()) print(f(n))

Как писать программу на экзамене

Главная ошибка — сразу лезть в код. Сначала нужно понять задачу. Вот проверенный алгоритм из четырёх шагов:

1 Прочитайте условие дважды — выделите: что дано, что найти, формат ввода/вывода.

2 Придумайте алгоритм на русском — запишите шаги в черновик без единой строчки кода.

3 Реализуйте и проверьте на примере из условия — если ответ совпал, переходите дальше.

4 Придумайте граничный случай — пустой файл, один элемент, все элементы одинаковые.

За правильно работающую программу с верным выводом на всех тестах дают 4 балла. За программу с правильной идеей, но ошибкой в выводе — 3 балла. Это всё равно хорошо.

Разбор реальной задачи ФИПИ

Возьмём задание из открытого банка ФИПИ 2025 года и разберём его пошагово.

📋 Условие задачи
Дан файл input.txt, содержащий натуральные числа, не превышающие 10 000. Количество чисел не превышает 10 000. Найдите два числа из файла, разность которых минимальна. Выведите эти два числа в порядке возрастания.

Наивное решение — перебрать все пары за O(n²) — может не уложиться во время. Оптимальное решение: отсортировать массив, тогда минимальная разность всегда у соседних элементов — O(n·log n).

Python — оптимальное решение
with open('input.txt') as f: nums = sorted(map(int, f.read().split())) min_diff = float('inf') ans = (0, 0) for i in range(len(nums) - 1): diff = nums[i + 1] - nums[i] if diff < min_diff: min_diff = diff ans = (nums[i], nums[i + 1]) print(ans[0], ans[1])

Результат: ✓ 4 балла — программа читает данные, применяет оптимальный алгоритм и выводит ответ в нужном формате.

Чек-лист перед сдачей

  • Программа читает данные из input.txt, а не из консоли (если требуется файл)
  • Вывод соответствует условию: пробелы, переносы строк, порядок чисел
  • Обработан случай, когда подходящих элементов нет
  • Нет лишних print() и отладочного вывода
  • Программа завершается без ошибок на всех тестах
